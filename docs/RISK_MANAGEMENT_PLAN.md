# FaceDoodle 项目风险管理计划

## 1. 风险识别 —— 风险分解结构 (RBS)

```
FaceDoodle 项目风险
├── 技术风险
│   ├── T1 外部服务依赖故障
│   ├── T2 AI 生成线程安全
│   ├── T3 跨平台兼容性
│   ├── T4 性能退化
│   └── T5 第三方库 API 变更
├── 工程风险
│   ├── E1 零测试覆盖
│   ├── E2 缺乏日志与可观测性
│   ├── E3 多进程通信故障
│   └── E4 代码腐化与回归
├── 资源风险
│   ├── R1 API Key 不可用
│   └── R2 硬件/GPU 资源不足
└── 产品风险
    ├── P1 用户体验断裂
    └── P2 数据丢失
```

## 2. 根因分析 —— 鱼骨图

### 问题陈述

**核心问题**: AI 贴纸生成失败或用户体验中断（应用不可用 / 功能降级 / 数据丢失）。

### 鱼骨图

![鱼骨图 — FaceDoodle 风险根因分析](鱼骨图.png)

### 根因分类与风险映射

| 鱼骨分支 | 根因 | 映射风险 | 表象 |
|---------|------|:------:|------|
| **外部依赖** | ComfyUI 无健康检查/重试/降级 | T1 | AI 生成超时，UI 无限等待 |
| | API Key 余额不可见，无提前告警 | R1 | 生成全部失败，用户困惑 |
| | GPU 环境下 AI 路径是唯一路径 | R2 | 无 GPU 时功能完全不可用 |
| **代码质量** | 曾 0 测试 → 每次修改都是盲目部署（现已补充 401 用例） | E1 | 新功能引入回归，老功能异常 |
| | `print()` 无持久化（现已全量迁移 logging） | E2 | 用户报 bug，开发者无法复现 |
| | 模块间直接引用（现已 Mixin + StickerRegistry 解耦） | E4 | 重构成本高，倾向于堆砌而非清理 |
| **运行环境** | `cv2.imread()` 不支持中文路径 | T3 | Windows 中文用户图片加载失败 |
| | MediaPipe/PySide6 API 不兼容旧版本 | T5 | 依赖升级后编译/运行时错误 |
| | 人脸检测无帧缓存 → 单帧丢失即崩溃 | P1 | 贴纸闪烁/消失，交互不可靠 |
| **进程通信** | 队列无大小上限 → 生产过剩 | E3 | 内存膨胀，消费者阻塞 |
| | 线程间共享可变状态无显式锁 | T2 | 偶发崩溃，难以复现 |
| | 渲染循环无帧率控制 → CPU 空转 | T4 | 高分辨率下交互延迟 |
| | index.json 无原子写入 → 断电风险 | P2 | 收藏/贴纸元数据丢失 |

## 3. 风险评估 —— FMEA (失效模式与影响分析)

采用 **RPN = S × O × D**（严重度 × 发生度 × 可探测度），每维 1–10 分。阈值：RPN ≥ 200 立即行动 | 100–199 本迭代处理 | < 100 监控。

| ID | 失效模式 | S | O | D | RPN | 等级 |
|----|---------|---|---|---|-----|------|
| E1 | 修改代码引入回归，UI 功能异常 | 7 | 9 | 9 | **567** | 致命 |
| T1 | ComfyUI 无响应，AI 生成挂起 | 9 | 7 | 6 | **378** | 致命 |
| E2 | 线上故障，无法定位根因 | 6 | 7 | 8 | **336** | 高 |
| T3 | 中文路径下 OpenCV 加载图片失败 | 8 | 7 | 4 | **224** | 高 |
| T2 | AI 结果写入与渲染读竞争，偶发崩溃 | 9 | 4 | 7 | **252** | 高 |
| R1 | API Key 欠费，生成全部失败 | 9 | 4 | 5 | **180** | 中 |
| E3 | 队列满导致消费者阻塞，画面冻结 | 10 | 2 | 6 | **120** | 中 |
| R2 | 无 GPU 时 AI 生成超时或 OOM | 8 | 4 | 3 | **96** | 中 |
| P2 | 贴纸文件损坏或 index.json 丢失 | 8 | 2 | 5 | **80** | 低 |
| T4 | 渲染帧率 < 10 FPS，交互卡顿 | 5 | 4 | 3 | **60** | 低 |
| P1 | 人脸短暂丢失，贴纸闪烁/消失 | 4 | 6 | 2 | **48** | 低 |
| T5 | PySide6/MediaPipe 大版本 API 破坏 | 7 | 1 | 2 | **14** | 低 |
| E4 | 模块间修改传播，需同步改多处 | 3 | 5 | 2 | **30** | 低 |
